# Steps to Configure GitHub insights

#### 1. Select the provider your app is configured with.

- Navigate to the Insights tab in the left sidebar of the SkyU Console.
- Click **GitHub** sidebar item.
- If your GitHub app is not configured, you will see the below box to configure insights.
- Click the **+ Configurations** button.

<Image
  src="/assets/images/insights/github/configurations.png"
  alt="Hello"
  width={1000}
  height={500}
/>

- Now you will be directed to Insights’ Settings page. If you have your app added already, you will see it listed under Github tab.

<Image
  src="/assets/images/insights/github/github-tab.png"
  alt="Hello"
  width={1000}
  height={500}
/>

- You will see your configured organization / workspace name, added apps and added custom apps for the insights configurations. ( Added apps through creating apps within the console and Custom apps through the “Add Custom Apps” button in Settings pages )
- Click **“Configure Github”** button to start the insights configurations.

#### 2. Configure GitHub data collection ( For Github connected apps )

- **Create your GitHub PAT (classic)** [How to get your PAT Token](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens) with the given permissions which are under **“Included Permissions”** and enter that PAT value in the “Pat Token Verification” tab’s Token input.

<Image
  src="/assets/images/insights/github/github-token.png"
  alt="Hello"
  width={1000}
  height={500}
/>

- After the validation is successful, You will be taken to the “Issues” tab and scroll down to click the “Next” button.
- Click the “Next” button in the “Pull Requests” tab.
- Click the “Next” button in the “Tags” tab.
- Click the “Create” button in the “Cross-Domain” tab.
- Once it's configured successfully, you will get the success message and “Done” button to complete the configurations.

<Image
  src="/assets/images/insights/github/github-final-configurations.png"
  alt="Hello"
  width={1000}
  height={500}
/>

- You will be redirected to the Settings page where you can see that the app is configured to use its insights.

<Image
  src="/assets/images/insights/github/github-configured.png"
  alt="Hello"
  width={1000}
  height={500}
/>

- Click the **“Sync With Devlake”** button to sync with the latest data in your configured repository.
- Now you can check the Gitbub sidebar item.

<Image
  src="/assets/images/insights/github/github-insights.png"
  alt="Hello"
  width={1000}
  height={500}
/>

#### 3. Custom GitHub app configuration

- Click **“+ Add Custom Apps”** button to create your custom app insights configuration in the “Insights Settings” page.

<Image
  src="/assets/images/insights/github/github-custom.png"
  alt="Hello"
  width={1000}
  height={500}
/>

- Click "Auth with GitHub" and after the popup, scroll down where you can select the GitHub account and repository.

<Image
  src="/assets/images/insights/github/github-custom-repo1.png"
  alt="Hello"
  width={1000}
  height={500}
/>
<Image
  src="/assets/images/insights/github/github-custom-repo2.png"
  alt="Hello"
  width={1000}
  height={500}
/>

- If the account you want to proceed with exists on the dropdown, you can select it and proceed to selecting its repositories.
- Otherwise if you want to connect with a new account, click "+ Create" and configure with your new account using the installation popup.

<Image
  src="/assets/images/insights/github/github-popup.png"
  alt="Hello"
  width={1000}
  height={500}
/>

- When you are configuring with a new account, you will be going through the steps of the initial process in which is mentioned at the first section of this documentation.

<Image
  src="/assets/images/insights/github/github-custom-config.png"
  alt="Hello"
  width={300}
  height={300}
/>

- Finally Click **"Create"** button to finish your configuration.
